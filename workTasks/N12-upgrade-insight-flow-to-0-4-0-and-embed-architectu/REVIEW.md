## Non-blocking findings (suggested follow-ups, not required for approval)

### NB-1 — `.claude/` is gitignored; slash-command extensions are per-developer

`.gitignore:14` lists `.claude`, so the role files containing `## Project Extensions` and the `@`-include wiring in `.claude/commands/*.md` are **not** committed. Source of truth is `taskflow.config.json`, but every developer who clones this repo must run `pnpm install && ./node_modules/.bin/insight-flow init` to materialize the role files, and (separately) hand-add `@.claude/roles/<ROLE>.md` references to their `.claude/commands/*.md` stubs to make slash-command invocations actually surface the extensions.

The commit body documents this; the spec was authored knowing the gap. Reasonable follow-ups:

- (a) Remove `.claude` from `.gitignore` (or change to `.claude/settings.local.json` only) so role files and command stubs are versioned.
- (b) Open a feature request upstream against `insight-flow` to make `init` patch `.claude/commands/*.md` with `@`-includes to the corresponding role file when `agents.extend` is non-empty.

Not in N12's scope; flagging for awareness.

### NB-2 — `workTasks/*.json` format violations are pre-existing

44 `workTasks/` files (across N03–N12) fail `npm run format:check`. The 3 shard JSON files I migrated (`master.json`, `tasks-N00-N09.json`, `tasks-N10-N19.json`) are in that set. `git stash && npm run format:check` against `main` reports 46 warnings → after applying N12, 44 warnings. No new regressions. Recommendation: add `workTasks` to `.prettierignore` OR commit a one-shot prettier sweep, separate from N12.

### NB-3 — `JSON.stringify` Unicode unescaping in the schema migration

The migration script (executed via `node -e`) used the default `JSON.stringify`, which emits literal Unicode characters (e.g. `—`, `→`, `↑`) rather than `—`/`→`/`↑` escapes the earlier `insight-flow` writes had used. The diff for `workTasks/tasks-N00-N09.json` includes ~10 such cosmetic substitutions that are content-identical and survive any JSON round-trip. No functional impact.

### NB-4 — `AGENT_ENFORCEMENT.md` is the upstream template text

The committed `AGENT_ENFORCEMENT.md` was generated verbatim from `insight-flow`'s template (12 lines: task-file-mutation enforcement + git/gh tool rule). It is referenced by `.claude/commands/arch-check.md` (`@AGENT_ENFORCEMENT.md`). Because `.claude/` is gitignored, that reference only resolves on machines that have re-run `insight-flow init` locally. The file itself is harmless to commit. If you'd rather not commit upstream-template text, deleting it after each `prompt-build` is also fine — the role files do not reference it.

### NB-5 — `/arch-check` smoke test was performed by hand, not as a slash command

The 7 checks in the contract were run inline by this review. Invoking `/arch-check` as a slash command in a fresh Claude session would exercise the same path through the agent skill file; that hasn't been verified end-to-end. Low risk because the slash file is a thin wrapper around grep + npm.
