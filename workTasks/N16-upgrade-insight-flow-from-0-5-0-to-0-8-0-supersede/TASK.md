# N16 — Upgrade insight-flow from 0.5.0 to 0.8.0 (supersedes N15)

**Type:** rework
**Priority:** medium
**Created:** 2026-05-27

## Problem

`insight-flow` is pinned at `0.5.0` in `package.json`. Upstream has shipped four releases since (`0.5.1`, `0.6.0`, `0.7.0`, `0.7.1`) and just published `0.8.0`. N15 had specced the 0.7.1 jump but was never implemented; this task supersedes it by going straight to `0.8.0` and includes whatever setup steps 0.8.0 ships with.

## Goal

1. Bump `insight-flow` in `package.json` from `0.5.0` to `0.8.0` and lock it in `pnpm-lock.yaml`.
2. Run `insight-flow init --force` to refresh project-side scaffolding (role files, `taskflow.config.json`, `.claude/hooks/*`) shipped with 0.8.0.
3. Run available migration / setup commands (`migrate`, `migrate-reviews`, `prompt-build --apply`, `install-activity-hook --force`) so tracker files, review files, and the enforcement block reflect the 0.8.0 schema.
4. Verify the local CLI (`./node_modules/.bin/insight-flow --version`) reports `0.8.0` and all everyday commands still work (`current`, `list`, `next-review`, `next-fix`, `next-change`, `stats`).
5. Close out N15 (PR #14) as superseded; library build / lint / typecheck stay green; no `src/` changes.

## Scope

### In scope

- `package.json` — bump `insight-flow` dep to `0.8.0`.
- `pnpm-lock.yaml` — regenerated by `pnpm install`.
- `taskflow.config.json` — update if `insight-flow init --force` modifies it.
- `.claude/roles/*.md` — re-emit if `init --force` updates role bodies; preserve `## Project Extensions` blocks (between `<!-- taskflow:extensions:start -->` and `<!-- taskflow:extensions:end -->`).
- `.claude/hooks/*.sh` — re-emit if `install-activity-hook --force` updates the activity hook; re-apply the local bash compatibility fix to `taskflow-classify.sh` (replace `case`-with-`|` with `if/elif grep -q`) if overwritten.
- `workTasks/*.json` — only touched by `migrate` / `migrate-reviews` if a tracker schema bump exists.
- `CLAUDE.md` (`## insight-flow` section) — minor edit if 0.8.0 introduces a command worth surfacing.
- Close N15 PR #14 (manual via `gh pr close 14 --comment "Superseded by N16 (#15)."` after this PR opens).

### Out of scope

- Anything under `src/` — tooling-only upgrade.
- The N14 consoleLogger module — already shipped.
- Cherry-picking features from 0.6.x / 0.7.x / 0.8.0 — install the whole `0.8.0`, accept it as a unit.
- Switching package manager — stay on `pnpm` (project rule).
- Removing the N15 task folder — leave the spec on disk for historical traceability.

### Breaking change

- **Possible.** Role-file regeneration, tracker-schema migration, or new mandatory commands shipped with 0.8.0 may alter on-disk shapes. Inspect every `git diff` after `init --force` and `migrate*` to confirm changes are intended; restore any `## Project Extensions` block that gets stomped.

## Implementation plan

1. **Pin and install 0.8.0**
   - Edit `package.json` → `"insight-flow": "0.8.0"`.
   - `pnpm install` to refresh `pnpm-lock.yaml`.
   - Confirm `./node_modules/.bin/insight-flow --version` prints `0.8.0`.

2. **Inspect the new command surface**
   - `./node_modules/.bin/insight-flow help` — diff against the 0.5.0 surface mentally. Note any new subcommands; if a previously-undocumented command appears, capture it in this TASK.md Notes before review.

3. **Re-emit project scaffolding**
   - Run `./node_modules/.bin/insight-flow init --force`.
   - For each modified file under `.claude/` and `taskflow.config.json`, eyeball `git diff` and confirm the `## Project Extensions` block (where present) is intact. If any block was stomped, restore it from `git show HEAD:<path>`.

4. **Run schema migrations**
   - `./node_modules/.bin/insight-flow migrate` — apply any tracker-shape migrations to `workTasks/master.json` and `workTasks/tasks-N00-N09.json` / `tasks-N10-N19.json`.
   - `./node_modules/.bin/insight-flow migrate-reviews` — idempotent in the current shape; should report no-op or apply 0.8.0's new shape.

5. **Refresh hooks and enforcement**
   - `./node_modules/.bin/insight-flow install-activity-hook --force` — ensure the PostToolUse activity hook matches the 0.8.0 contract. If it overwrites `.claude/hooks/taskflow-classify.sh`, re-apply the `if/elif grep -q` rewrite (see git log for the original fix during N14 setup).
   - `./node_modules/.bin/insight-flow prompt-build --apply` — regenerate the enforcement block (`AGENT_ENFORCEMENT.md` and any per-role enforcement footer).

6. **Smoke-test the CLI**
   - `./node_modules/.bin/insight-flow current` — should return N16.
   - `./node_modules/.bin/insight-flow list` — every task lists; status fields readable.
   - `./node_modules/.bin/insight-flow next` / `next-review` / `next-fix` / `next-change` — return reasonable picks.
   - `./node_modules/.bin/insight-flow stats` — completes without error.
   - Launch the dashboard (`./node_modules/.bin/insight-flow`, default port 6007) and confirm task list + N14 history render.

7. **Run library quality gates**
   - `npx tsc --noEmit` — zero errors.
   - `npm run lint` — zero warnings (`--max-warnings 0`).
   - `npm run format:check` — no new files dirty (pre-existing project-wide debt is acceptable; do not touch unrelated files).
   - `npm run build` — `dist/` produced.

8. **Close N15 and ship N16**
   - After implementation pushes and PR opens, close N15 PR #14 with a comment pointing to N16's PR.
   - Optionally mark N15 abandoned in the tracker via `insight-flow status --id N15 --status abandoned` (or leave it as-is for history).
