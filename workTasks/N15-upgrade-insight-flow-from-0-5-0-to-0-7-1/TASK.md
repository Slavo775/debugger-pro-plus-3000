# N15 — Upgrade insight-flow from 0.5.0 to 0.7.1

**Type:** rework
**Priority:** medium
**Created:** 2026-05-26

## Problem

`insight-flow` is currently pinned at `0.5.0` in `package.json`. Upstream has shipped `0.5.1`, `0.6.0`, `0.7.0`, and `0.7.1`. We need to pick up the latest stable (`0.7.1`) and run any setup / migration steps that ship with it (new role files, hooks, enforcement blocks, tracker schema updates) so the local install stays current.

## Goal

1. Bump `insight-flow` in `package.json` from `0.5.0` to `0.7.1` and lock it in `pnpm-lock.yaml`.
2. Run `insight-flow init` (with `--force` if needed) to refresh any project-side scaffolding (role files, `taskflow.config.json`, `.claude/hooks/*`) that ships with 0.7.1.
3. Run available migration commands (`migrate`, `migrate-reviews`, `prompt-build --apply`, `install-activity-hook`) so tracker files, review files, and the enforcement block reflect the 0.7.1 schema.
4. Verify the local CLI (`./node_modules/.bin/insight-flow --version`) reports `0.7.1` and all everyday commands still work (`current`, `list`, `next-review`, `next-fix`, `next-change`).
5. Library build and lint stay green — no regressions in `src/` or in the consumer-facing package.

## Scope

### In scope

- `package.json` — bump `insight-flow` dependency to `0.7.1`.
- `pnpm-lock.yaml` — regenerated by `pnpm install`.
- `taskflow.config.json` — update if `insight-flow init` modifies it (project-side config schema may have grown).
- `.claude/roles/*.md` — re-emit if `insight-flow init` updates role files; preserve `## Project Extensions` (taskflow:extensions:start/end) blocks intact.
- `.claude/hooks/*.sh` — re-emit if `insight-flow install-activity-hook` updates the activity hook; preserve any local syntax fixes (see N14's taskflow-classify.sh fix).
- `workTasks/*.json` — touched only by `migrate` / `migrate-reviews` if a schema bump is present.
- `CLAUDE.md` (`## insight-flow` section) — minor edit if new commands or workflow changes warrant it.

### Out of scope

- Application code under `src/` — no source-file changes; this is a tooling upgrade.
- The N14 consoleLogger module — already shipped.
- Removing or renaming existing role files; only `--force` re-emit if the new version asks for it.
- Switching package manager — stay on `pnpm` per project rules.
- Cherry-picking features from 0.6.x or 0.7.x — install the whole 0.7.1, accept it as a unit.

### Breaking change

- **Possible.** Role-file regeneration or tracker-schema migration may alter on-disk shapes. If `insight-flow init --force` would overwrite the `## Project Extensions` block, restore it from git before committing. Treat any `taskflow.config.json` schema bump as the new baseline.

## Implementation plan

1. **Pin and install the new version**
   - `pnpm add -D insight-flow@0.7.1` (or edit `package.json` then `pnpm install`).
   - Verify `./node_modules/.bin/insight-flow --version` prints `0.7.1`.

2. **Inspect what changed in the bundled assets**
   - `./node_modules/.bin/insight-flow help` — confirm the command surface; note any new subcommands.
   - List role-file diffs by running `insight-flow init --force` against a temp copy if needed; OR just run it on the project and rely on `git diff` to spot changes.

3. **Re-emit project scaffolding**
   - Run `./node_modules/.bin/insight-flow init --force` to refresh `.claude/roles/*.md`, `taskflow.config.json`, and any other scaffolding.
   - Inspect each modified role file in `git diff` — the body may have new instructions; the `## Project Extensions` block MUST stay intact (between `<!-- taskflow:extensions:start -->` and `<!-- taskflow:extensions:end -->`).

4. **Run schema migrations**
   - `./node_modules/.bin/insight-flow migrate` — apply tracker-shape migrations to `workTasks/master.json` and `workTasks/tasks-*.json`.
   - `./node_modules/.bin/insight-flow migrate-reviews` — split inline review/incident state if a side-file model arrived in 0.6/0.7 (already split for N14; should be idempotent).

5. **Refresh hooks and enforcement**
   - `./node_modules/.bin/insight-flow install-activity-hook --force` — ensure the PostToolUse activity hook matches the 0.7.1 contract. If the hook overwrites `.claude/hooks/taskflow-classify.sh`, re-apply the bash compatibility fix made earlier (replace `case`-with-`|` patterns with `if/elif grep -q` — see git log for the original fix).
   - `./node_modules/.bin/insight-flow prompt-build --apply` — regenerate the enforcement block (`AGENT_ENFORCEMENT.md` or whatever the 0.7.1 emitter writes).

6. **Smoke-test the CLI**
   - `./node_modules/.bin/insight-flow current` — should still return N15.
   - `./node_modules/.bin/insight-flow list` — every task with no regressions.
   - `./node_modules/.bin/insight-flow next` / `next-review` / `next-fix` / `next-change` — return reasonable picks.
   - `./node_modules/.bin/insight-flow stats` — completes without error.

7. **Run library gates**
   - `npx tsc --noEmit` — zero errors.
   - `npm run lint` — zero warnings (per `--max-warnings 0`).
   - `npm run format:check` — no new files dirty (pre-existing project-wide debt is acceptable).
   - `npm run build` — `dist/` produced.

8. **Document the bump**
   - Update `CLAUDE.md` `## insight-flow` section IF the new version surfaces commands worth calling out for future contributors.
   - Note any breaking-change asks in `TASK.md` Notes before review.
